# Jello Knight Balance Plan

## 목적

이 문서는 `Jello Knight`의 핵심 성장 스탯과 적 운영값을 `최대 레벨 5` 기준으로 정리한 밸런스 기준 문서다.

- 기준 1: 현재 구현 중인 플레이 감각을 크게 해치지 않을 것
- 기준 2: 모바일 웹에서도 복잡한 계산 없이 유지 가능한 수치 체계일 것
- 기준 3: 초반은 단순하고, 후반은 확실히 강해지는 성장 체감을 만들 것

이 문서의 `Lv1`은 시작 기본값, `Lv5`는 최대 성장값이다.

## 공통 설계 원칙

- 한 번의 레벨업 선택으로 스탯 1단계만 상승
- 각 항목은 `Lv5` 이상 상승하지 않음
- 오브 계열은 `지속 화력`
- 폭탄 계열은 `확률성 광역 폭딜`
- 생존 계열은 `실수 완충 장치`

## 크리티컬 규칙

크리티컬 데미지 스탯은 단독 수치가 아니라 `고정 크리티컬 확률`과 함께 사용한다.

- 오브 크리티컬 확률: `8%`
- 폭탄 크리티컬 확률: `12%`
- 크리티컬 데미지 표의 수치는 `기본 피해 대비 배수`
- 예시: 오브 피해 2, 오브 크리 데미지 1.8x 이면 크리티컬 시 피해 `3.6`

## 방어력 규칙

방어력은 단순 퍼센트 감소형으로 관리한다.

- 최종 피해 계산식: `받는 피해 = 원피해 x (1 - 방어력)`
- 너무 높은 무적감 방지를 위해 최대 `24%`까지만 허용

## 100 웨이브 운영 원칙

`Jello Knight`는 외부적으로는 `무한 점수형 생존 런`처럼 보이되, 내부적으로는 `100개 웨이브 단위의 스폰/난이도 테이블`로 운영하는 방식을 권장한다.

현재 설계 방향은 이 문서를 기준으로 `목표 처치 수 기반 wave 1 ~ 100 상승`, `웨이브 기반 스폰 비율`, `5단계 비주얼 장애물 확장` 구조로 맞춘다.

핵심 이유:

- 플레이어는 점수를 계속 올리는 흐름을 유지할 수 있음
- 내부 밸런스는 `웨이브 1 ~ 100` 기준으로 촘촘하게 조정 가능
- 특정 웨이브 구간마다 `적 조합`, `출현 빈도`, `최대 동시 수`, `장애물 압박`을 안정적으로 설계 가능
- 시간보다 `사냥 성과`가 웨이브 진행을 만든다는 점에서 플레이 감각이 더 명확함
- 넓은 맵에서도 `이번 웨이브를 밀어냈다`는 감각을 만들기 쉽다

### 기본 구조

| 항목 | 설계값 |
| --- | --- |
| 최대 웨이브 | `100` |
| 웨이브 진행 방식 | 목표 처치 수 기반 |
| 웨이브 클리어 조건 | 해당 웨이브의 `목표 처치 수` 달성 |
| 전멸 요구 여부 | `없음`, 마지막 적 정리 강제하지 않음 |
| 웨이브 전환 유예 | `1.2초 ~ 1.8초` |
| 사이클 종료 후 | wave 100 유지 상태로 무한 생존 지속 또는 재시작 규칙 선택 가능 |

### 권장 해석

- `wave 1 ~ 20`: 조작 적응과 첫 성장 확보 구간. 적 수는 적지만 명확히 쓸어 담는 감각을 준다.
- `wave 21 ~ 40`: 근거리 밀집과 첫 원거리 진입 구간. 목표 수가 늘며 웨이브 진행 속도가 살아난다.
- `wave 41 ~ 60`: 원거리 견제와 첫 엘리트 압박 구간. 사냥 효율과 회피 운영을 동시에 요구한다.
- `wave 61 ~ 80`: 성장 상태가 부족하면 밀리기 시작하는 중후반 구간. 웨이브 처치 목표가 확실히 무거워진다.
- `wave 81 ~ 100`: 높은 처치 수와 복합 조합을 밀어내는 엔드게임 구간. 생존력보다 운영 완성도가 중요해진다.

### wave 1 체감 목표

- 적이 너무 빠르게 몰리지 않아야 함

- 첫 웨이브는 `6 ~ 10킬` 전후에서 안정적으로 정리되는 것이 이상적
- 첫 레벨업은 `wave 1 종료 전후` 혹은 `wave 2 초반`에 도달하는 정도가 가장 자연스럽다

## 웨이브 100 기준 최종 압박 목표

`wave 100`은 불합리한 즉사 구간이 아니라, `충분히 성장한 빌드만 안정적으로 유지 가능한 상한선`이어야 한다.

### wave 100 목표 감각

- 근거리 적은 많이 나오지만, 패턴 자체는 읽을 수 있어야 함
- 원거리 적은 수가 아니라 위치 선정으로 압박해야 함
- 엘리트는 화면을 지배하되, 피격 회피 가능성이 있어야 함
- 풀업 기준 플레이어는 `고득점 유지`가 가능해야 함

## 구현 권장 로직

현재 `danger tier 1 ~ 5` 구조는 프로토타입으로는 충분하지만, `세밀한 스폰 밸런스`를 위해서는 `waveIndex` 중심 구조로 넘어가는 것이 좋다.

### 권장 전환 순서

1. `waveIndex`, `waveKillCount`, `waveTargetKillCount`를 별도 상태로 관리
2. 적이 죽을 때마다 `waveKillCount += 1`
3. `waveKillCount >= waveTargetKillCount`가 되면 다음 웨이브 예약
4. `dangerTier`는 제거하거나, `waveIndex`에서 파생된 시각 연출용 값으로만 축소
5. 스폰 간격/최대 수/비율/원거리 활성/엘리트 활성은 전부 `waveIndex` 기준으로 계산
6. 문서 표는 `10웨이브 단위 밴드`로 유지하고, 구현은 밴드 내부 선형 보간 사용

### 계산식 예시

```ts
bandIndex = Math.floor((waveIndex - 1) / 10)
bandProgress = ((waveIndex - 1) % 10) / 9

waveTargetKills = lerp(band.killStart, band.killEnd, bandProgress)

if (waveKillCount >= waveTargetKills) {
  queueNextWave(1400)
}
```

이후 각 밴드의 시작값과 끝값을 선형 보간하면, 100개 웨이브를 일일이 하드코딩하지 않고도 충분히 촘촘한 난이도 설계가 가능하다.

### 잔여 적 처리 권장안

완전 전멸형은 쓰지 않고, 목표 처치 수 달성 순간 다음 웨이브로 넘어가는 구조를 권장한다.

- 기존 필드 적은 즉시 삭제하지 않음
- 다음 웨이브 전환 유예 동안 신규 스폰만 잠깐 멈춤
- 전환 이후에는 새 웨이브 규칙으로 스폰 재개
- 잔여 적이 과도하게 누적될 경우에만 `동시 수 상한`으로 자연 정리

이 구조가 가장 중요한 이유는 `마지막 1마리 찾기 스트레스`를 없애면서도 `내가 웨이브를 밀어냈다`는 감각은 유지할 수 있기 때문이다.

### 엘리트 대쉬 패턴 현재 구현 기준

공통 원칙:

- 상시 돌진이 아니라 `내부 쿨다운 + 랜덤 트리거` 기반
- 대쉬 전 짧은 예고 시간 부여
- 대쉬 중에는 방향을 크게 틀지 않고 현재 타깃 방향으로 짧게 돌진
- 저사양 모바일을 고려해 복잡한 경로 계산보다 `짧은 직선 burst 이동`으로 설계

| 타입 | 대쉬 예고 | 대쉬 속도 배수 | 대쉬 지속 시간 | 대쉬 내부 쿨다운 | 패턴 의도 |
| --- | --- | --- | --- | --- | --- |
| `🦖` | `0.55초` | `4.2x` | `1.10초` | `4.8초 ~ 6.2초` | 무겁게 준비했다가 강하게 밀고 들어오는 돌진 |
| `🦂` | `0.32초` | `4.0x` | `0.56초` | `2.6초 ~ 3.6초` | 빠르게 파고들며 긴장감을 주는 곡선 대쉬 |
| `🕷️` | `0.44초` | `1.9x` | `0.34초` | `4.2초 ~ 5.4초` | 돌진 후 거미줄 장판을 남겨 이동 동선을 제어 |

대쉬 체감 목표:

- `🦖`는 피하기는 가능하지만 맞으면 크게 위험한 패턴
- `🦂`는 잦고 빠르지만 개별 한 번의 압박은 `🦖`보다 덜 무거운 패턴
- `🕷️`는 대쉬형이 아니라 지속 추적 + 거미줄 투척 기반 패턴
- 플레이어가 이동을 멈추고 있으면 엘리트가 계속 직선 추적만 하며 굳어 보이지 않도록 리듬을 흔들어주는 역할

### `🕷️` 거미줄 장판 규칙 현재 구현 기준

`🕷️`는 이동하며 거미줄을 포물선 투척하고, 필드에 거미줄 장판을 남긴다.

공통 원칙:

- 장판은 `저사양 모바일`을 고려해 수를 제한하되 현재 구현은 `동시 최대 5개`
- 장판은 `직접 피해`보다 `이동 둔화` 중심
- 장판 위에 오래 머무를수록 위험해지지 않게, 효과는 단순 감속으로 통일
- 장판은 플레이어만 둔화시키고 적 이동에는 영향 없음

| 항목 | 값 |
| --- | --- |
| 장판 반경 | `72` |
| 지속 시간 | `자동 삭제 없음` |
| 감속률 | `기본 감속 배수 0.78` |
| 플레이어 접촉 감속 | `3초 동안 50% 감소` |
| 동시 최대 수 | `5개` |
| 생성 타이밍 | `3000ms 간격 포물선 투척` |

설계 의도:

- `🕷️`는 즉시 큰 피해보다 `이동 경로를 꼬이게 만드는 엘리트`
- 장애물과 겹쳐질 때 지나치게 답답해지지 않도록 장판 수는 적게 유지
- `🦖`, `🦂`가 몸으로 압박할 때 `🕷️`는 공간을 봉쇄하는 역할로 분리

### 운영 메모

- 첫 엘리트는 `wave 11 ~ 25` 구간에서 `🦖` 단독으로 노출해 학습시킨다.
- `🦖`는 오래 버티는 대신 느리고, `🦂`는 덜 단단하지만 빠르게 압박, `🕷️`는 장판으로 운영을 흔든다.
- 실제 구현 시 현재 단일 엘리트 구조를 `타입 테이블 + 부가 패턴 테이블` 기반으로 전환하는 것을 권장

- `orbitCountRef` 추가
- 오브 배치 각도는 `360 / 개수`
- 현재 단일 오브 계산을 배열 기반으로 일반화

### 오브 크리티컬 데미지

- 충돌 시 `Math.random() <= 0.08`
- 치명타면 `orbitDamage x orbitCritMultiplier`

### 폭탄 드롭 확률

- 현재 `BOMB_TRIGGER_CHANCE`를 레벨별 수치로 치환
- `bombDropChanceRef`로 관리

### 폭탄 드롭 간격

- 현재 `BOMB_TRIGGER_INTERVAL_MS`를 레벨별 수치로 치환
- `bombDropIntervalRef`로 관리

### 폭탄 크리티컬 데미지

- 폭탄 폭발 판정 시 `Math.random() <= 0.12`
- 치명타면 `bombDamage x bombCritMultiplier`

### 생명력

- `PLAYER_MAX_HP` 상수 대신 `playerMaxHpRef`
- HP 바와 회복 아이템 최대치 계산 모두 같은 값을 참조

### 방어력

- contact / projectile / elite 피해 계산 직전
- `reducedDamage = Math.max(1, Math.round(baseDamage * (1 - defenseRate)))`

### 이동 속도

- 현재 `PLAYER_BASE_SPEED` 대신 `playerMoveSpeedRef`
- 조이스틱 감도는 유지하고 실제 이동량만 증가

## 추천 업그레이드 그룹 구조

레벨업 카드 풀은 아래 3그룹으로 관리하는 것이 좋다.

- 공격: 오브 공격력, 오브 개수, 오브 스핀 속도, 오브 범위, 오브 크리티컬 데미지
- 폭탄: 폭탄 드롭 확률, 폭탄 드롭 간격, 폭탄 공격 반경, 폭탄 크리티컬 데미지
- 생존: 생명력, 방어력, 젤로 이동 스피드

추천 규칙:

- 한 번의 3지선다에 같은 그룹 카드가 2개를 넘지 않게 제한
- 폭탄은 `bomb_unlock` 이후에만 관련 카드 노출
- 이미 `Lv5`에 도달한 스탯은 카드 풀에서 제거

## 메모

- 현재 표는 최근 코드 구현값 기준으로 다시 동기화한 상태다.
- 적 수치, 접촉 반경, 오브 타격 반경, 엘리트 대쉬 계수는 `enemyBehaviors.ts`를 우선 기준으로 본다.
- 플레이어 성장, 폭탄, 웨이브 전환, 거미줄 공통 상수는 `constants.ts`를 우선 기준으로 본다.
- 오브 개수 증가는 성능 영향이 있으므로 `Lv5 최대 4개`를 권장 상한으로 둔다.
