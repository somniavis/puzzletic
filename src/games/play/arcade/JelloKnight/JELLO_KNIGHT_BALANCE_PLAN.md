# Jello Knight Balance Plan

## 목적

이 문서는 `Jello Knight`의 현재 코드 기준 핵심 성장 스탯, 웨이브 구조, 적 운영값을 정리한 최신 밸런스 기준 문서다.

- 기준 1: 현재 구현 중인 플레이 감각을 크게 해치지 않을 것
- 기준 2: 모바일 웹에서도 복잡한 계산 없이 유지 가능한 수치 체계일 것
- 기준 3: 초반은 단순하고, 후반은 확실히 강해지는 성장 체감을 만들 것

기본 기준:

- 대부분의 스킬은 `Lv1 ~ Lv5`
- `오브 개수`만 `Lv1 ~ Lv4`
- 세트 기반 웨이브는 현재 실제 코드에 연결되어 있음
- 세트별 적 HP/데미지 스케일링도 현재 실제 코드에 연결되어 있음

## 문서 상태

이 문서는 아래 원칙으로 읽는다.

- `현재 구현 기준` 또는 `현재 적용값`으로 적힌 항목은 실제 코드 반영 완료
- 별도 표시가 없는 수치표도 현재 코드 기준 동기화를 우선
- 후반 누적 성장처럼 아직 반영되지 않은 항목은 별도 `보류안`으로 분리해 둔다

## 공통 설계 원칙

- 한 번의 레벨업 선택으로 스탯 1단계만 상승
- 대부분 항목은 `Lv5`까지, `오브 개수`만 `Lv4`까지 상승
- 오브 계열은 `지속 화력`
- 폭탄 계열은 `확률성 광역 폭딜`
- 생존 계열은 `실수 완충 장치`

## 크리티컬 규칙

크리티컬 데미지 스탯은 단독 수치가 아니라 `고정 크리티컬 확률`과 함께 사용한다.

- 오브 크리티컬 확률: `8%`
- 폭탄 크리티컬 확률: `12%`
- 크리티컬 데미지 표의 수치는 `기본 피해 대비 배수`
- 예시: 오브 피해 2, 오브 크리 데미지 1.8x 이면 크리티컬 시 피해 `3.6`

## 방어력 규칙

방어력은 단순 퍼센트 감소형으로 관리한다.

- 최종 피해 계산식: `받는 피해 = 원피해 x (1 - 방어력)`
- 너무 높은 무적감 방지를 위해 현재 구현 상한은 `22%`까지만 허용

## 100 웨이브 운영 원칙

`Jello Knight`는 외부적으로는 `무한 점수형 생존 런`처럼 보이되, 내부적으로는 `100개 웨이브 단위의 스폰/난이도 테이블`로 운영한다.

현재 설계 방향은 이 문서를 기준으로 `목표 처치 수 기반 wave 1 ~ 100 상승`, `10웨이브 1세트 반복 강화`, `세트 내부 장애물 단계 확장` 구조로 맞춘다.

핵심 이유:

- 플레이어는 점수를 계속 올리는 흐름을 유지할 수 있음
- 내부 밸런스는 `웨이브 1 ~ 100` 기준으로 촘촘하게 조정 가능
- 특정 웨이브 구간마다 `적 조합`, `출현 빈도`, `최대 동시 수`, `장애물 압박`을 안정적으로 설계 가능
- 시간보다 `사냥 성과`가 웨이브 진행을 만든다는 점에서 플레이 감각이 더 명확함
- 넓은 맵에서도 `이번 웨이브를 밀어냈다`는 감각을 만들기 쉽다

### 기본 구조

| 항목 | 설계값 |
| --- | --- |
| 최대 웨이브 | `100` |
| 웨이브 진행 방식 | 목표 처치 수 기반 |
| 웨이브 클리어 조건 | `일반 적 처치 목표` 달성, 엘리트 웨이브는 `엘리트 처치 목표` 추가 충족 |
| 전멸 요구 여부 | `없음`, 마지막 적 정리 강제하지 않음 |
| 웨이브 전환 유예 | `1400ms` |
| wave 100 이후 | 현재 문서는 `wave 1 ~ 100` 밸런스 기준까지 정의 |

### 운영 구간 해석

- `wave 1 ~ 20`: 조작 적응과 첫 성장 확보 구간. 적 수는 적지만 명확히 쓸어 담는 감각을 준다.
- `wave 21 ~ 40`: 근거리 밀집과 첫 원거리 진입 구간. 목표 수가 늘며 웨이브 진행 속도가 살아난다.
- `wave 41 ~ 60`: 원거리 견제와 첫 엘리트 압박 구간. 사냥 효율과 회피 운영을 동시에 요구한다.
- `wave 61 ~ 80`: 성장 상태가 부족하면 밀리기 시작하는 중후반 구간. 웨이브 처치 목표가 확실히 무거워진다.
- `wave 81 ~ 100`: 높은 처치 수와 복합 조합을 밀어내는 엔드게임 구간. 생존력보다 운영 완성도가 중요해진다.

## 세트 기반 최종 웨이브 계획

현재 기준으로 가장 적합한 방향은 `100웨이브를 개별 설계`하는 것이 아니라, `10웨이브 1세트`를 완성도 있게 만들고 그 세트를 10번 반복하며 난이도만 점층 강화하는 구조다.

핵심 원칙:

- 한 세트 안에 이미 `근접형 / 원거리형 / 엘리트형`을 모두 등장시킨다
- 세트가 뒤로 갈수록 조합은 더 복합적으로, 전장은 더 답답하게 만든다
- 세트 반복 시에는 `조합 자체`보다 `강도`, `밀도`, `장애물 단계`, `스폰 방향 수`를 높인다
- 저사양 대응을 위해 새 시스템을 늘리지 않고 기존 `적 / 스폰 / 장애물 / 거미줄 / 엘리트 로직`을 재조합한다

한 줄 요약:

`세트 안에서 게임의 모든 맛을 보여주고, 세트 반복은 그 맛을 더 강하게 만든다`

## 세트 운영 구조

### 기본 프레임

- `10웨이브 = 1세트`
- `100웨이브 = 10세트`
- 각 세트는 같은 골격을 유지하되 점점 더 세고 더 복합적으로 강화된다

### 세트 내부 체감 흐름

- `1 ~ 2`: 기본 적응과 라인 읽기
- `3 ~ 5`: 복합 전투 진입
- `6 ~ 7`: 엘리트 게이트 전투
- `8 ~ 10`: 요새 압박, 특수전, 클라이맥스

## 세트 내 장애물 오픈 규칙

현재 구현된 내부/외부 성곽 슬롯 규칙은 세트 안에서 아래 순서로 확장시키는 것이 가장 자연스럽다.

| 웨이브 | 장애물 단계 | 전장 감각 |
| --- | --- | --- |
| `1 ~ 2` | 내부 모서리만 활성 | 가장 단순한 적응 전장 |
| `3 ~ 5` | 내부 모서리 + 내부 면 3면 개방 | 통로 전투 시작 |
| `6 ~ 8` | 내부 구조 유지 + 외부 모서리 추가 | 외곽 압박 확대 |
| `9 ~ 10` | 내부 구조 유지 + 외부 면 일부까지 확장 | 세트 후반 공성전 감각 |

즉 세트 안에서도 전장은 `단순 -> 내부 통로 -> 외곽 압박 -> 확장 전장` 순서로 성장한다.

## 적 조합 운영 원칙

각 웨이브는 아래 구조로 읽히도록 설계한다.

- `주연 적 1 ~ 2종`
  해당 웨이브의 핵심 위협
- `보조 적 2 ~ 3종`
  전투 밀도와 리듬 형성
- `소량 적 0 ~ 1종`
  변주를 주는 조미료 역할

중요한 점:

- 적이 많이 나오는 것보다 `무슨 웨이브인지 읽히는 것`이 더 중요하다
- 세트 후반으로 갈수록 주연 적 수와 보조 적 수가 동시에 늘어난다
- `8 ~ 10` 구간은 복합도가 가장 높지만, 여전히 주연 위협은 분명해야 한다

## 세트 웨이브 전환 조건

세트 기반 웨이브는 `일반 적 처치 목표`와 `엘리트 처치 목표`를 분리해서 관리하는 것이 가장 명확하다.

### 기본 규칙

- 일반 웨이브:
  `일반 적 처치 목표`만 달성하면 다음 웨이브로 넘어간다
- 엘리트 포함 웨이브:
  `일반 적 처치 목표 달성`과 `엘리트 적 최소 1회 이상 처치`를 모두 만족해야 다음 웨이브로 넘어간다

### 현재 내부 상태

- `waveKillTarget`
- `waveEliteKillTarget`
- `waveKillCount`
- `waveEliteKillCount`

### 현재 판정식

```ts
const clearedWave =
  waveKillCount >= waveKillTarget
  && waveEliteKillCount >= waveEliteKillTarget;
```

이 구조가 좋은 이유:

- 엘리트 웨이브가 시간만 지나서 넘어가는 느낌을 막아준다
- 엘리트가 등장한 웨이브의 존재감이 분명해진다
- 일반 처치 목표와 엘리트 목표를 따로 조정할 수 있어 밸런스 수정이 쉬워진다

### 추가 운영 원칙

- `8 Fortress Crush`처럼 엘리트가 2종 이상 등장하더라도 기본 목표는 `엘리트 1킬`로 유지한다
- `10 Climax Hunt`처럼 모든 적이 출현해도, 세트 마무리의 핵심은 `총력전 감각`이므로 `엘리트 1킬 + 높은 일반 처치 목표` 구조가 적합하다

## 8~10 웨이브 운영 보정 원칙

후반 3웨이브는 복합도가 높아지는 대신, `주연 위협`이 흐려지지 않게 아래 원칙을 둔다.

- `8 Fortress Crush`
  `🦖` 또는 `🦂` 중 한쪽을 주연으로 두고 다른 한쪽은 보조처럼 운영
- `9 Web Cross`
  반드시 `🕷️`가 주연이어야 하며, `🦖`는 보조 축으로만 사용
- `10 Climax Hunt`
  모든 적 출현 가능으로 두되, 세트마다 `주연 2종`은 고정해 개성을 유지

예시:

- 세트1 클라이맥스: `🦂 + 👁️‍🗨️`
- 세트2 클라이맥스: `🦖 + 🧿`
- 세트3 클라이맥스: `🕷️ + 🦂`

## 세트 반복 시 강화 방식

세트가 반복될수록 아래 요소만 점층 강화한다.

- 적 HP 상승
- 스폰 간격 단축
- 동시 최대 수 증가
- 원거리 비중 증가
- 엘리트 등장 빈도 증가
- 장애물 확장 단계 상승
- 외곽 스폰 비중 증가
- 완충감은 줄이고 클라이맥스 밀도는 강화

즉 `같은 세트 구조`라도 뒤로 갈수록 더 빠르고, 더 많이, 더 복합적으로 압박하게 된다.

## 세트1 예상 레벨업 점검

현재 구현 수치를 기준으로 보면, `1세트 1 ~ 10`은 후반 목표 처치 수가 높아 보이더라도 `레벨업 가속`이 함께 일어나는 구조다.

### 기준 수치

- XP 오브 기본 획득값: `17`
- 레벨업 필요 XP:
  `100 + (현재 레벨 - 1) * 45`
- XP 드롭 기본 확률:
  `42.5%`

즉 레벨업 요구량은 점진적으로 증가하지만, `세트 중반 이후 엘리트 처치`가 들어오면서 획득 XP 기대값도 함께 크게 상승한다.

### 드롭 기준 기대 XP

현재 드롭 구조를 기대값으로 단순 환산하면 대략 아래처럼 해석할 수 있다.

| `🧿` | `1` | 약 `10.6` |
| `👁️‍🗨️` | `2` | 약 `27.2` |
| `🦖` | `5` | 약 `127.5` |
| `🦂` | `5` | 약 `102` |
| `🕷️` | `5` | 약 `114.8` |

핵심 포인트:

- 일반 적만 잡는 초반에는 레벨업이 완만하게 진행된다
- `wave 6` 이후 엘리트 1킬 조건이 들어오면서 XP 획득량이 크게 뛴다
- 따라서 `세트1 후반`은 일반 적 처치 목표가 높아도 성장으로 충분히 따라잡을 가능성이 높다

추가 기준:

- 엘리트 3종 `🦖`, `🦂`, `🕷️`는 모두 `5드롭` 기준을 사용한다
- 기대 XP 차이는 `드롭 수`가 아니라 `엘리트별 xpValue` 차이에서 발생한다

## 세트별 적 능력치 스케일링 기준

현재 스킬 구조를 기준으로 보면 플레이어는 `세트 1`에서 빠르게 성장하고, `세트 2 ~ 3`에서 핵심 빌드가 거의 완성되며, 이후는 `완성된 빌드 운용 능력`이 더 중요해진다.

그래서 세트 스케일링은 아래 원칙으로 잡는 것이 가장 안정적이다.

- 적 `체력`은 세트가 올라갈수록 확실히 증가
- 적 `공격력`은 체력보다 훨씬 완만하게 증가
- `이동속도`는 세트 공통 고정에 가깝게 두고, 위협은 조합/밀도/장애물로 만든다
- `엘리트`는 이미 패턴 자체가 강하므로 HP만 너무 과하게 키우지 않는다
- `세트 6` 이후부터는 수치 상승보다 `복합 조합`, `스폰 방향`, `장애물 단계`가 주 위협이 되어야 한다

## 장애물 슬롯 운영 기준

장애물은 `고정 슬롯 집합`을 미리 배치한 뒤, 웨이브의 비주얼 단계에 따라 일부만 활성화하는 구조를 기준으로 한다.

### 구조 원칙

- 내부 요새 구조는 `좌상/우상 가로벽`, `좌우 세로벽 2쌍`, `좌하/우하 가로벽`을 기본 축으로 유지
- 내부 중앙 보강용 소형 벽은 `상중`, `하중`에 각각 `96 x 74` 슬롯으로 추가
- 외곽 슬롯은 `코너`, `상중/하중`, `좌중/우중`으로 분산 배치
- 외곽 중앙 슬롯은 `긴 벽 + 짧은 벽` 조합으로 읽히도록 비대칭 배치
  - 상중: `긴 벽 -> 짧은 벽`
  - 하중: `짧은 벽 -> 긴 벽`
  - 좌중: `긴 벽(위) -> 짧은 벽(아래)`
  - 우중: `짧은 벽(위) -> 긴 벽(아래)`

### 웨이브 연동 활성 기준

장애물은 이제 `10웨이브 1세트 내부 단계 확장` 방식으로 동작한다. 즉 세트 안에서 구조가 점차 확장되고, 다음 세트로 넘어가면 같은 골격을 유지한 채 배치 패턴과 압박 강도만 강화된다.

| 세트 내 구간 | 활성 슬롯 기준 |
| --- | --- |
| `wave 1 ~ 2` | 내부 벽 `모서리 슬롯`만 활성 |
| `wave 3 ~ 5` | 내부 벽 `모서리 + 면 슬롯 3면` 활성 |
| `wave 6 ~ 8` | 내부 구조 유지 + 외부 벽 `모서리 슬롯` 추가 |
| `wave 9 ~ 10` | 내부/외부 모서리 유지 + 외부 벽 `면 슬롯 3면` 활성 |

### 면 슬롯 선택 규칙

- 내부 면 슬롯은 `북/동/남/서` 4면 중 매 배치 주기마다 `3면`만 열린다
- 외부 면 슬롯도 동일하게 `4면 중 3면`만 열린다
- 남는 `1면`은 통로 역할을 하도록 비워둔다
- 배치 주기는 `세트 내부 2웨이브 단위`로 완만하게 바뀌게 두는 것이 적절하다

예시:

- `set 1 / wave 3 ~ 4`: 내부 3면 개방 패턴 A
- `set 1 / wave 5`: 내부 3면 개방 패턴 B
- `set 1 / wave 9`: 외부 3면 개방 패턴 A
- `set 1 / wave 10`: 외부 3면 개방 패턴 B

### 디버그 슬롯 표시 기준

- 디버그 슬롯 표시는 `실제 활성 장애물`과 분리 관리한다
- 디버그가 켜져 있을 때는 `전체 슬롯 집합`을 모두 표시해 배치를 검수한다
- 실제 충돌/벽 활성화는 웨이브 단계 기준으로만 계산한다
- 배포 전에는 슬롯 점선, 슬롯 라벨, 중심점 표시는 비활성화한다

- `🦂`는 잦고 빠르지만 개별 한 번의 압박은 `🦖`보다 덜 무거운 패턴

- 플레이어가 이동을 멈추고 있으면 엘리트가 계속 직선 추적만 하며 굳어 보이지 않도록 리듬을 흔들어주는 역할

### `🕷️` 거미줄 장판 규칙

`🕷️`는 이동하며 거미줄을 포물선 투척하고, 필드에 거미줄 장판을 남긴다.

공통 원칙:

- 장판은 `저사양 모바일`을 고려해 수를 제한하되 현재 구현은 `동시 최대 5개`
- 장판은 `직접 피해`보다 `이동 둔화` 중심
- 장판 위에 오래 머무를수록 위험해지지 않게, 효과는 단순 감속으로 통일
- 장판은 플레이어만 둔화시키고 적 이동에는 영향 없음

| 항목 | 값 |
| --- | --- |
| 장판 반경 | `72` |
| 지속 시간 | `자동 삭제 없음` |
| 감속률 | `기본 감속 배수 0.78` |
| 플레이어 접촉 감속 | `3초 동안 50% 감소` |
| 동시 최대 수 | `5개` |
| 생성 타이밍 | `3000ms 간격 포물선 투척` |

설계 의도:

- `🕷️`는 즉시 큰 피해보다 `이동 경로를 꼬이게 만드는 엘리트`
- 장애물과 겹쳐질 때 지나치게 답답해지지 않도록 장판 수는 적게 유지
- `🦖`, `🦂`가 몸으로 압박할 때 `🕷️`는 공간을 봉쇄하는 역할로 분리

### 엘리트 운영 기준

- 첫 엘리트는 `1세트 wave 6`에서 `🦖` 단독으로 노출해 학습시킨다.
- `🦖`는 오래 버티는 대신 느리고, `🦂`는 덜 단단하지만 빠르게 압박, `🕷️`는 장판으로 운영을 흔든다.

## 성장 / 업그레이드 기준

### 업그레이드 상한 정책

스킬 업그레이드 팝업에서는 별 개수로 최대 레벨을 직접 노출하지 않고, 내부 로직에서만 각 스킬 최대 레벨을 유지한다.

핵심 원칙:

- 스킬 팝업은 `아이콘 + 이름 + 상승 수치`만 간결하게 노출
- 내부 로직은 기능별 최대 레벨을 계속 유지
- 최대 레벨에 도달한 스킬은 카드 풀에서 제거
- 구조를 크게 바꾸는 기능은 상한 이후 추가 누적을 허용하지 않음
- 수치형 화력/생존 축의 후반 누적 성장은 아직 코드 미적용 상태다

## 현재 업그레이드 그룹 구조

레벨업 카드 풀은 현재 아래 3그룹으로 관리한다.

- 공격: 오브 공격력, 오브 개수, 오브 스핀 속도, 오브 범위, 오브 크리티컬 데미지
- 폭탄: 폭탄 드롭 확률, 폭탄 드롭 간격, 폭탄 공격 반경, 폭탄 크리티컬 데미지
- 생존: 생명력, 방어력, 젤로 이동 스피드

현재 규칙:

- 한 번의 3지선다에 같은 그룹 카드가 2개를 넘지 않게 제한
- 폭탄은 `bomb_unlock` 이후에만 관련 카드 노출
- 각 스킬 최대 레벨에 도달한 스탯은 카드 풀에서 제거

## 기준 파일 우선순위

- 적 수치, 접촉 반경, 오브 타격 반경, 엘리트 대쉬 계수: `enemyBehaviors.ts`
- 플레이어 성장, 폭탄, 웨이브 전환, 거미줄 공통 상수: `constants.ts`
- 세트 구조, 웨이브 타입, 세트별 전투 배수: `waveConfig.ts`
- 적 생성, 드롭, 업그레이드 카드 구성: `helpers.ts`
- 실시간 판정과 최종 적용: `useJelloKnightGame.ts`
- 오브 개수는 실제 구현 기준 `4단계 최대 4개`
