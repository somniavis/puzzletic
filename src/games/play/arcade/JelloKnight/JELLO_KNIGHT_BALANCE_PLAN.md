# Jello Knight Balance Plan

## 목적

이 문서는 `Jello Knight`의 핵심 성장 스탯을 `최대 레벨 5` 기준으로 설계한 밸런스 초안이다.

- 기준 1: 현재 구현 중인 플레이 감각을 크게 해치지 않을 것
- 기준 2: 모바일 웹에서도 복잡한 계산 없이 유지 가능한 수치 체계일 것
- 기준 3: 초반은 단순하고, 후반은 확실히 강해지는 성장 체감을 만들 것

이 문서의 `Lv1`은 시작 기본값, `Lv5`는 최대 성장값이다.

## 공통 설계 원칙

- 한 번의 레벨업 선택으로 스탯 1단계만 상승
- 각 항목은 `Lv5` 이상 상승하지 않음
- 오브 계열은 `지속 화력`
- 폭탄 계열은 `확률성 광역 폭딜`
- 생존 계열은 `실수 완충 장치`

## 크리티컬 규칙

크리티컬 데미지 스탯은 단독 수치가 아니라 `고정 크리티컬 확률`과 함께 사용한다.

- 오브 크리티컬 확률: `8%`
- 폭탄 크리티컬 확률: `12%`
- 크리티컬 데미지 표의 수치는 `기본 피해 대비 배수`
- 예시: 오브 피해 2, 오브 크리 데미지 1.8x 이면 크리티컬 시 피해 `3.6`

## 방어력 규칙

방어력은 단순 퍼센트 감소형으로 관리한다.

- 최종 피해 계산식: `받는 피해 = 원피해 x (1 - 방어력)`
- 너무 높은 무적감 방지를 위해 최대 `24%`까지만 허용

## 100 웨이브 운영 원칙

`Jello Knight`는 외부적으로는 `무한 점수형 생존 런`처럼 보이되, 내부적으로는 `100개 웨이브 단위의 스폰/난이도 테이블`로 운영하는 방식을 권장한다.

현재 구현 방향도 이 문서를 기준으로 `구간형 시간 규칙 기반 wave 1 ~ 100 상승`, `웨이브 기반 스폰 비율`, `5단계 비주얼 장애물 확장` 구조로 맞춘다.

핵심 이유:

- 플레이어는 점수를 계속 올리는 흐름을 유지할 수 있음
- 내부 밸런스는 `웨이브 1 ~ 100` 기준으로 촘촘하게 조정 가능
- 특정 웨이브 구간마다 `적 조합`, `출현 빈도`, `최대 동시 수`, `장애물 압박`을 안정적으로 설계 가능

### 기본 구조

| 항목 | 설계값 |
| --- | --- |
| 최대 웨이브 | `100` |
| 웨이브 진행 방식 | 시간 기반 자동 상승 |
| 웨이브 길이 | `1~30: 15초 / 31~60: 13초 / 61~80: 10초 / 81~100: 8초` |
| 총 한 사이클 길이 | 약 `1200초`, 즉 `20분` |
| 사이클 종료 후 | wave 100 유지 상태로 무한 생존 지속 또는 재시작 규칙 선택 가능 |

### 권장 해석

- `wave 1 ~ 20`: 조작 적응과 첫 성장 확보 구간. 웨이브 길이가 길어 초반 학습 여유를 준다.
- `wave 21 ~ 40`: 근거리 밀집이 본격화되는 구간. 원거리 진입 전 회피 감각을 익히게 한다.
- `wave 41 ~ 60`: 원거리 견제와 첫 엘리트 압박이 체감되는 구간. 템포는 유지하되 웨이브 길이는 아직 충분하다.
- `wave 61 ~ 80`: 전투 밀도가 빠르게 올라가는 중후반 구간. 성장 상태 점검이 본격적으로 시작된다.
- `wave 81 ~ 100`: 가장 짧은 웨이브 구간으로, 최대 압박을 지속적으로 유지하는 엔드게임 구간.

## 웨이브 100 기준 최종 압박 목표

- 근거리 적은 많이 나오지만, 패턴 자체는 읽을 수 있어야 함

- 중간 성장 기준 플레이어는 `wave 70 ~ 85` 부근부터 밀리기 시작하고, `wave 90+`부터는 생존 유지가 도전적으로 느껴지는 것이 이상적

## 구현 권장 로직

현재 `danger tier 1 ~ 5` 구조는 프로토타입으로는 충분하지만, `세밀한 스폰 밸런스`를 위해서는 `waveIndex` 중심 구조로 넘어가는 것이 좋다.

### 권장 전환 순서

1. `elapsedMs`에서 `구간형 웨이브 시간 규칙`을 기준으로 `waveIndex` 계산
2. `dangerTier`는 제거하거나, `waveIndex`에서 파생된 시각 연출용 값으로만 축소
3. 스폰 간격/최대 수/비율/원거리 활성/엘리트 활성은 전부 `waveIndex` 기준으로 계산
4. 문서 표는 `10웨이브 단위 밴드`로 유지하고, 구현은 밴드 내부 선형 보간 사용

### 계산식 예시

```ts
if (waveIndex <= 30) durationMs = 15000
else if (waveIndex <= 60) durationMs = 13000
else if (waveIndex <= 80) durationMs = 10000
else durationMs = 8000

bandIndex = Math.floor((waveIndex - 1) / 10)
bandProgress = ((waveIndex - 1) % 10) / 9
```

이후 각 밴드의 시작값과 끝값을 선형 보간하면, 100개 웨이브를 일일이 하드코딩하지 않고도 충분히 촘촘한 난이도 설계가 가능하다.

## 원거리 적 2종 초안

사용자 기준표에서 `접촉 판정 반경`이 공통/개별 항목에 모두 들어 있었는데, 실제 플레이 밸런스상 타입 차이를 만드는 데 중요하므로 `개별 항목`으로 정리한다.

## 엘리트 적 2종 초안

엘리트 적은 `강제 처치형 보스`가 아니라 `피할 수도 있고, 잡으면 큰 보상을 얻는 고위험 표적`으로 유지한다.

| 동시에 존재 | `1마리` |
| 크기 | `122` |
| 첫 엘리트 출현 기준 | danger tier 4 이후 `18초` |

### 엘리트 2종 최종 수치표

| 타입 | 이모지 | 역할 | HP | 이동속도 기본 | 접촉 판정 반경 | 접촉 데미지 | 스폰 간격 | 처치 보상 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 타입 A | `🦖` | 둔중 돌진형 | `28` | `68` | `54` | `12` | `20000ms` | `XP 드롭값 60`, 고등급 동물 구조 오브 다수 |

`스폰 간격`은 첫 출현 이후 다음 엘리트 출현 예약 시간으로 사용한다.

설계 의도:

- `🦖`는 존재감이 큰 메인 엘리트
- `🦂`는 더 자주 런 흐름을 흔드는 템포 엘리트
- 두 타입은 `체급과 속도` 차이만 봐도 성격이 바로 느껴지도록 분리

### 엘리트 대쉬 패턴 초안

엘리트 적은 기본 추적만 하지 않고 `비정기적 대쉬`를 사용한다.

공통 원칙:

- 상시 돌진이 아니라 `내부 쿨다운 + 랜덤 트리거` 기반
- 대쉬 전 짧은 예고 시간 부여
- 대쉬 중에는 방향을 크게 틀지 않고 현재 타깃 방향으로 짧게 돌진
- 저사양 모바일을 고려해 복잡한 경로 계산보다 `짧은 직선 burst 이동`으로 설계

| 타입 | 대쉬 예고 | 대쉬 속도 배수 | 대쉬 지속 시간 | 대쉬 내부 쿨다운 | 패턴 의도 |
| --- | --- | --- | --- | --- | --- |
| `🦖` | `0.55초` | `2.3x` | `0.42초` | `4.8초 ~ 6.2초` | 무겁게 준비했다가 강하게 밀고 들어오는 돌진 |
| `🦂` | `0.32초` | `2.8x` | `0.26초` | `2.6초 ~ 3.6초` | 빠르게 파고들며 긴장감을 주는 날카로운 대쉬 |

대쉬 체감 목표:

- `🦖`는 피하기는 가능하지만 맞으면 크게 위험한 패턴
- `🦂`는 잦고 빠르지만 개별 한 번의 압박은 `🦖`보다 덜 무거운 패턴
- 플레이어가 이동을 멈추고 있으면 엘리트가 계속 직선 추적만 하며 굳어 보이지 않도록 리듬을 흔들어주는 역할

### 운영 메모

- danger tier 개편 전 초기 출현 비율은 `🦖 60% / 🦂 40%` 정도가 안정적
- `🦖`는 오래 버티는 대신 느리고, `🦂`는 덜 단단하지만 빠르게 압박
- 실제 구현 시 현재 단일 엘리트 구조를 타입 테이블 기반으로 전환하는 것을 권장

## 구현 권장 방식

### 오브 개수

- `orbitCountRef` 추가
- 오브 배치 각도는 `360 / 개수`
- 현재 단일 오브 계산을 배열 기반으로 일반화

### 오브 크리티컬 데미지

- 충돌 시 `Math.random() <= 0.08`
- 치명타면 `orbitDamage x orbitCritMultiplier`

### 폭탄 드롭 확률

- 현재 `BOMB_TRIGGER_CHANCE`를 레벨별 수치로 치환
- `bombDropChanceRef`로 관리

### 폭탄 드롭 간격

- 현재 `BOMB_TRIGGER_INTERVAL_MS`를 레벨별 수치로 치환
- `bombDropIntervalRef`로 관리

### 폭탄 크리티컬 데미지

- 폭탄 폭발 판정 시 `Math.random() <= 0.12`
- 치명타면 `bombDamage x bombCritMultiplier`

### 생명력

- `PLAYER_MAX_HP` 상수 대신 `playerMaxHpRef`
- HP 바와 회복 아이템 최대치 계산 모두 같은 값을 참조

### 방어력

- contact / projectile / elite 피해 계산 직전
- `reducedDamage = Math.max(1, Math.round(baseDamage * (1 - defenseRate)))`

### 이동 속도

- 현재 `PLAYER_BASE_SPEED` 대신 `playerMoveSpeedRef`
- 조이스틱 감도는 유지하고 실제 이동량만 증가

## 추천 업그레이드 그룹 구조

레벨업 카드 풀은 아래 3그룹으로 관리하는 것이 좋다.

- 공격: 오브 공격력, 오브 개수, 오브 스핀 속도, 오브 범위, 오브 크리티컬 데미지
- 폭탄: 폭탄 드롭 확률, 폭탄 드롭 간격, 폭탄 공격 반경, 폭탄 크리티컬 데미지
- 생존: 생명력, 방어력, 젤로 이동 스피드

추천 규칙:

- 한 번의 3지선다에 같은 그룹 카드가 2개를 넘지 않게 제한
- 폭탄은 `bomb_unlock` 이후에만 관련 카드 노출
- 이미 `Lv5`에 도달한 스탯은 카드 풀에서 제거

## 메모

- 현재 구현값과 완전히 일치하지 않는 항목은 `목표 설계값`이다.
- 특히 폭탄 드롭 확률은 현재 체감상 다소 잦아 보일 수 있으므로, 실제 구현 시 `Lv1 22%`부터 시작하는 방향을 권장한다.
